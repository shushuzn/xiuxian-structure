## 🎮 v1.2.0 — 互动小说引擎

> **数据驱动的修仙互动叙事** —— 把 11 体系 yaml 变成运行时世界数据库，由状态机引擎驱动分支剧情。

### 核心组件

#### `scripts/interactive.py`（547 行）
- **`World`** — 从 `data/*.yaml` 构建内存世界，支持 `lookup` / `find_by_id` / `render_template`
- **`Story`** — 解析 `stories/*.md`（自创轻量 DSL：`## 节点 <id> [(type)]` + YAML 字段块）
- **`Engine`** — 状态机 + 条件求值 + data 初始化
- **`State`** — 玩家属性 + 标志位 + `check(expr)` 条件表达式求值
- **CLI**：交互模式 / headless 模式（CI 用）
- **程序化 API**：`Engine.play()` / `Engine.step()` / `Engine.save()` / `Engine.load()`

#### 状态机制
- `data` 块：节点进入时设置状态
  - 普通 key（`灵石: 5`）：总是覆盖
  - `?` 前缀 key（`?灵石: 5`）：**仅当未设置时初始化**（保护玩家存档）
- **条件分支**：`if: 灵石 >= 3` / `flag.拜师` / 支持 `and` / `or` / `not`
- **模板占位符**：`{realms.炼气期.lifespan}` 自动从 yaml 取值

### 设计亮点

**1. 数据驱动**（一致性的根本保障）
剧情里写 `{realms.炼气期.lifespan}` 会被自动从 yaml 拉值 —— 剧情永远跟数据一致。

**2. 条件分支 + 状态机**
- `if: 灵石 >= 3` 过滤可用选项
- `flag.拜师` 检查标志位
- `set: { 境界: 筑基初期 }` 选中后修改状态
- 灵石 = 5 → 入李家结局；灵石 = 1 → 自动跳到坊市结局（条件过滤生效）

**3. 故事校验（CI）**
新增 `Verify all stories are well-formed` 步骤：
- 起点节点存在
- 所有跳转目标存在（BFS 验证）
- 至少 1 个 ending
- 无孤儿节点

### CI 增强

| 步骤 | 验证内容 |
|---|---|
| `Run validation` | 知识库结构 / 链接 / yaml / mermaid / 索引 |
| `Run export (smoke test)` | export.py 工具可执行 |
| `Run interactive engine (smoke test)` | 互动引擎可跑 demo |
| `Verify all stories are well-formed` | **新增** stories/ 连通性 + 完整性 |
