## 🤖 v1.3.0 — LLM 协作

> **把仓库从「静态资源」推向「AI 协作中心」** —— 让大语言模型半自动扩展互动故事节点。

### 核心组件

#### `scripts/generate_node.py`（~340 行）
- **`build_world_summary`** — 从 `data/*.yaml` 自动生成 11 体系 + 关键 ID 摘要（喂 LLM 用）
- **`build_prompt`** — 严格 prompt 模板（7 条硬性规则）
- **`call_llm`** — OpenAI 兼容 API 调用（支持 `LLM_BASE_URL` / `LLM_MODEL` 环境变量）
- **`NodeValidator`** — 5 项校验
  - id 不与已有冲突（ERROR）
  - 解析成功（ERROR）
  - 所有 `goto` 目标存在（ERROR）
  - `refs` 指向的文件存在（WARN）
  - 数值范围：灵石 0-10000 / 声望 0-100（WARN）
- **失败自动反馈重试** — 3 轮内把错误传给 LLM 修复

#### 工作流（3 阶段）

```
[1. 你写需求]              1 段自然语言（10-50 字）
[2. 脚本自动]              喂 yaml + 现有故事 → LLM → 解析 + 校验 → 重试或写盘
[3. 你审核 + 接入]         把 goto 加到合适节点
```

### 新增交付物

- **`examples/feed-to-llm.md`** — 工作流指南（快速开始 / 7 条最佳实践 / 与其他工具的协作）
- **`examples/generated/夜探禁地.md`** — mock LLM 输出（2 个节点 + NodeValidator 校验通过）

### 设计亮点

**1. 数据驱动（防幻觉）**
- 自动从 `data/*.yaml` 生成 11 体系 + 关键 ID 摘要
- LLM 写 `refs: 符箓体系/火球符.md` 时**知道**这个文件存在
- LLM 写 `境界: 筑基期` 时**知道**寿元 200 / 突破需筑基丹

**2. 严格 prompt（结构化输出）**
- 7 条硬性规则（id 命名 / 必填字段 / 跳转目标 / 数值一致）
- 错误信息直接进入重试 prompt

**3. NodeValidator 5 项校验**
- 防御 LLM 常见错误：id 冲突 / 解析失败 / goto 不存在
- 不阻塞警告：refs 不存在（用户后续可补）、数值范围超界

**4. 失败自动反馈**
- ERROR 进入重试 prompt（3 轮）
- WARN 只在 stderr 提示
- 重试用尽后最后一次输出 dump 到 stderr（人工编辑后重跑）
