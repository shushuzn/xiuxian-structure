## 🌐 v1.4.0 — Web UI

> **把互动小说从 CLI/curl 推向浏览器** —— FastAPI 后端 + 暗色主题 SPA 前端。

### 核心组件

#### `scripts/web_app.py`（~310 行 · FastAPI 后端）
- 8 个 REST 端点：health / list / get / session create+get+choice+restart
- Pydantic 模型：NodeView / StoryView / SessionView / ChoiceRequest
- 内存 sessions（重启丢失，简单）
- 静态文件挂载（`/static` + `/play` SPA 入口）

#### `interactive/web/`（3 文件 · 零依赖前端）
- **`index.html`** — SPA 骨架（3 块布局：故事列表 / 主区 / 状态面板）
- **`style.css`** — 暗色修仙主题（衬线字体 + 暖金/冷蓝调色板）
- **`app.js`** — vanilla JS（6 个 API 函数）

### 8 个 REST 端点

| Method | Path | 用途 |
|---|---|---|
| GET | `/health` | 健康检查 |
| GET | `/` | 列出所有 stories |
| GET | `/story/{id}` | 故事元数据 |
| POST | `/story/{id}/session` | 创建会话（可选 `initial_state`） |
| GET | `/session/{sid}` | 读当前状态 |
| POST | `/session/{sid}/choice` | 选选项（`choice_index`） |
| POST | `/session/{sid}/restart` | 重启 |
| GET | `/play` | 🆕 前端入口 |
| GET | `/play/{story_id}` | 🆕 直达故事 |

### 设计亮点

**1. API-only 架构**
- server 持 state（session_id → Engine）
- client 只持 sid，刷新无状态丢失
- 易扩展：未来加 WebSocket / SSE 实时推流

**2. 零依赖前端**
- 单 HTML + 单 CSS + 单 JS，共 ~17KB（gzip 后 ~5KB）
- 零构建工具（不需要 npm/webpack/vite）
- 任何静态服务器都能跑

**3. 暗色修仙主题**
- 衬线字体（Noto Serif SC）+ 暖金 accent（仙家符箓）+ 冷蓝（灵气）
- 卡片 hover / 选项左 border 动画 / ending 渐变背景
- 移动端响应式（< 900px 折叠为单列）

**4. 严格 CI 测试**
- 8 个 REST 端点 + 4 个 UI 端点 + 1 个 HTML 挂载点检查
- 真实 server + curl 模式（避免 TestClient 兼容问题）

### 体验

```bash
pip install -r requirements.txt
python3 scripts/web_app.py
# → http://localhost:8000/play
# → API 文档: /docs
```

### CI 流水线

| 步骤 | 验证内容 |
|---|---|
| `Run validation` | 知识库 5 项校验 |
| `Run export (smoke test)` | export.py 可执行 |
| `Run interactive engine (smoke test)` | 互动引擎可跑 |
| `Verify all stories are well-formed` | stories/ 连通性 + 完整性 |
| `Verify LLM-generated examples are parseable` | NodeValidator 5 项 |
| **`Verify Web API endpoints`** | 🆕 12 个端点 + SPA 挂载点 |
