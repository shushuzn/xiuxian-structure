# 文档模板（DO NOT DELETE）

本仓库所有体系文档必须遵守的统一模板。新建 .md 时复制本文件，按需删除不适用的章节。

---

## 基础章节（任何 .md 都必须有）

```markdown
# {体系/对象名}

## 定义
（一句话定义 + 在世界观中的定位）

## 构成要素
（按维度拆解。每个维度一个 ### 小节。）

### {维度 1}
- ...

### {维度 2}
- ...

## 关联
（本文件涉及到的其他体系文件，列出相对路径）

- 上游：影响本体系的其他体系
- 下游：本体系影响的其他体系
- 同级：同级别相关条目

## 引用
（可选）引用来源，如《凡人修仙传》某卷某章节，或其他修仙设定参考。
```

## 类型化模板

### A. 境界/阶段类（如 炼气期.md、筑基期.md）

必含小节：
- ## 定位
- ## 等级划分（初期/中期/后期/巅峰/大圆满）
- ## 寿命上限
- ## 核心能力（神识/法力/特殊能力）
- ## 突破条件（法力/心境/外物/风险）
- ## 限制（做不到的事情）
- ## 关联

### B. 资源/对象类（如 灵根.md、灵气.md、丹药.md、功法.md、法器.md）

必含小节：
- ## 定义
- ## 构成要素
  - ### 品级（如果是分层对象）
  - ### 属性/分类
  - ### 关键机制
- ## 关联

### C. 索引/总览类（如 索引.md、README.md）

必含小节：
- ## 作用说明
- ## 体系清单
- ## 关联关系图

---

## 命名规范

- 文件名：用中文，与体系/对象同名。如 `灵根.md`、`筑基期.md`。
- 标题：`# ` 后直接跟名称，不用 "XXX 定义/介绍" 这种后缀。
- 小节标题：用中文，不用英文。
- 列表：用 `- ` 不用 `* ` 或 `1. `。

## 链接规范

- 引用其他 .md 用相对路径：`../丹药体系/丹药.md`
- 锚点引用：`../丹药体系/丹药.md#突破类`
- GitHub 风格链接：`[筑基丹](../丹药体系/丹药.md)`

## 数据规范

每个体系在 `data/{体系}.yaml` 必须有对应结构化数据，作为：
1. 程序读取的权威数据源
2. CI 校验的参考

YAML schema 在 `docs/SCHEMA.md` 中定义。

## 校验

CI 会检查：
1. 每个 .md 包含必备章节
2. `关联` 章节里写的链接文件真实存在
3. data/*.yaml 通过 schema 校验
4. mermaid 图语法正确

## 反例

❌ 不要：
- 一句话概况整个体系（如 "## 概述\n修仙的第一步。"）
- 把所有维度塞进一个大段落
- 在多个文件重复定义同一个概念（应当有唯一权威源）

✅ 要：
- 每章 3-7 个要点
- 维度分明，结构对称
- 跨体系引用走"关联"章节
